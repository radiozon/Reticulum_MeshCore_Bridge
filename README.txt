================================================================================
                    RNS <-> MeshCore Bridge
================================================================================

Мост для интеграции сетей Reticulum/LXMF (MeshChat) и MeshCore (тестировался
на базе Heltec V3). Позволяет обмениваться сообщениями между пользователями 
Reticulum и участниками MeshCore-сети, а также публиковать новости в формате 
NomadNet.

1. ВОЗМОЖНОСТИ
================================================================================

- Двунаправленный обмен сообщениями между Reticulum/LXMF (MeshChatX) и MeshCore
- Публикация новостей в формате .mu (NomadNet) из любого канала
- Авторизация по паролю для управления мостом из Reticulum
- Ники - удобное отображение имён пользователей.
- Автоматический маяк - периодическая отправка статуса в эфир сети MeshCore
- Логирование всех сообщений с ротацией и сжатием
- Автоаннонсы в сети Reticulum
- Готовые скрипты запуска для удобного управления

================================================================================
2. ТРЕБОВАНИЯ
================================================================================

2.1. Системные зависимости:

sudo apt update
sudo apt install python3 python3-pip screen git unzip -y

2.2. Python-пакеты:

pip3 install rns lxmf meshcore

ВАЖНО:
- Убедитесь, что meshcore-cli доступен в системе и настроен для работы с LoRa
- Для работы с NomadNet необходим установленный Reticulum и LXMF клиент.
- Рекомендуется использовать Python 3.11 или новее.

================================================================================
3. УСТАНОВКА
================================================================================

3.1. Установка из архива (NomadNet):

1. Скачайте архив MeshCore_RNGit.zip со страницы проекта в NomadNet
2. Распакуйте архив:

unzip MeshCore_RNGit.zip -d ~/RNS_MeshCore
cd ~/RNS_MeshCore

3. Сделайте скрипты исполняемыми:

chmod +x RMesh_final.py
chmod +x start.sh
chmod +x run_bridge.sh
chmod +x start_terminal.sh

3.2. Настройка после установки:

1. Создайте конфигурационный файл:

cp bridge_config.ini.example bridge_config.ini
# или создайте вручную
nano bridge_config.ini

2. Отредактируйте bridge_config.ini (см. раздел "Настройка")

3. Создайте директорию для хранения данных:

mkdir -p ~/.meshcore_bridge_storage

================================================================================
4. НАСТРОЙКА
================================================================================

4.1. Полный список параметров bridge_config.ini:

ОСНОВНЫЕ:
---------
port            string    Порт MeshCore-устройства          /dev/ttyUSB0
channel         string    Имя канала в MeshCore             Public
key             string    Ключ шифрования (не используется)  8b3387e9...
name            string    Имя моста в Reticulum             MeshBridge
announce        int       Интервал анонсов в минутах        30
auth_password   string    Пароль для авторизации            4777
baudrate        int       Скорость UART                     115200

НОВОСТИ:
--------
news            on/off    Включить публикацию новостей      on
news_file       string    Путь к файлу новостей .mu         /home/user/.../news.mu
max_news        int       Максимальное количество записей   50
news_hash       string    Хеш личности для ссылок           d0e9b8129f6b...

МАЯК:
-----
beacon_enabled  on/off    Включить автоматический маяк      on
beacon_interval int       Интервал маяка в секундах         7200
beacon_text     string    Текст маяка                       📡 Мост активен

ЛОГИРОВАНИЕ:
------------
log_enabled     on/off    Включить логирование              on
log_retention_days int    Дни хранения логов                7

ПРИМЕЧАНИЕ: Мост совместим с MeshChatX и стандартным MeshChat.
Путь к файлу новостей:
- MeshChatX: ~/.reticulum-meshchatx/storage/page_nodes/.../pages/news.mu
- MeshChat:  ~/.reticulum-meshchat/storage/page_nodes/.../pages/news.mu


================================================================================
5. КОМАНДЫ
================================================================================

5.1. Команды из Reticulum (MeshChat):

/pass <пароль>    Авторизация в мосте          /pass 4777
/stop             Завершить сессию             /stop
/news <текст>     Опубликовать новость         /news Важное объявление!
/name <ник>       Установить ник               /name Alice

5.2. Команды из MeshCore:

/news <текст>     Опубликовать новость из эфира   /news Встреча завтра в 15:00

================================================================================
6. ФУНКЦИОНАЛ НОВОСТЕЙ
================================================================================

6.1. Требования к файлу новостей:

Файл .mu должен содержать специальный маркер:

`B48b`Ffff
📌  Последние сообщения из эфира
`b`f

6.2. Формат записи:

│ 
│ `F6be` 📅 2024-01-15 14:30:00`f
│ `F60f` 👤 @Username`f `Ffff `b`f через`f MeshCore `Ff00 Public`f
│ `Ffff➥  `b`f Текст новости`f
│ 

6.3. Ссылка на новости:

📡 Смотреть: [<news_hash>:/page/news.mu]

================================================================================
7. АВТОМАТИЧЕСКИЙ МАЯК
================================================================================

Периодически отправляет указанный текст в канал MeshCore.

Пример настройки:

beacon_enabled = on
beacon_interval = 7200
beacon_text = 📡 Мост RNS активен. Для связи используйте MeshChat.

ВАЖНО: Интервал указывается в СЕКУНДАХ. 

================================================================================
8. ЛОГИРОВАНИЕ
================================================================================

Все сообщения логируются в структурированном формате JSON.

8.1. Формат записи лога:

{
  "ts": "2024-01-15T14:30:00.123456",
  "dir": "from_meshcore",
  "from": "Alice",
  "from_hash": "a1b2c3d4e5f6...",
  "text": "Привет всем!",
  "text_hash": "abcdef1234567890",
  "channel": "Public",
  "channel_hash": "1234567890abcdef",
  "to": null,
  "ver": "v1.0.0"
}

8.2. Ротация логов:

- Логи хранятся с указанием даты: bridge_YYYYMMDD.log
- Старые логи автоматически сжимаются в .gz
- Удаление по истечении log_retention_days

================================================================================
9. ЗАПУСК
================================================================================

9.1. Скрипты запуска (входят в архив):

Файл                    Описание
----------------------  --------------------------------------------------------
RMesh_final.py          Основной скрипт моста
start.sh                Интерактивный запуск с выводом в консоль
run_bridge.sh           Фоновый запуск (для screen/cron)
start_terminal.sh       Запуск в новом окне терминала (для GUI)

----------------------  --------------------------------------------------------

Как использовать:
mv start.sh
chmod +x start.sh

9.3. Запуск в интерактивном режиме:

./start.sh

или

python3 RMesh_final.py

9.4. Запуск в фоновом режиме (рекомендуется):

./run_bridge.sh

9.5. Запуск через screen:

# Создание сессии
screen -dmS bridge ./run_bridge.sh

# Подключение к сессии
screen -r bridge

# Выход из сессии (без остановки)
Ctrl+A, затем D

9.6. Запуск через systemd (для автозапуска):

Создайте файл /etc/systemd/system/meshcore-bridge.service:

[Unit]
Description=RNS MeshCore Bridge
After=network.target

[Service]
Type=simple
User=orangepi
WorkingDirectory=/home/orangepi/RNS_MeshCore
ExecStart=/home/orangepi/RNS_MeshCore/run_bridge.sh
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

Затем активируйте:

sudo systemctl daemon-reload
sudo systemctl enable meshcore-bridge
sudo systemctl start meshcore-bridge

9.7. Проверка работы:

ps aux | grep RMesh_final
tail -f ~/.meshcore_bridge_storage/logs/bridge_*.log
sudo systemctl status meshcore-bridge

9.8. Остановка:

# Если запущен через screen
screen -X -S bridge quit

# Если запущен вручную
Ctrl+C

# Если через systemd
sudo systemctl stop meshcore-bridge

Мост НЕ ТРЕБУЕТ ДОСТУПА В ИНТЕРНЕТ и работает полностью в локальных сетях:
- Reticulum может работать через TCP, UDP или LoRa

================================================================================
15. УСТРАНЕНИЕ НЕПОЛАДОК
================================================================================

15.1. Мост не видит устройство:

Проверьте:
ls -la /dev/ttyUSB*
dmesg | grep tty

Права доступа:
sudo chmod 666 /dev/ttyUSB0

Добавление пользователя в группу dialout:
sudo usermod -a -G dialout $USER
# Требуется перезагрузка или перелогин

15.2. Новости не добавляются:

Проверьте:
1. Файл существует и доступен для записи:
ls -la /path/to/news.mu

2. В файле есть маркер:
`B48b`Ffff
📌  Последние сообщения из эфира
`b`f

3. Параметр news = on в конфиге

15.3. Сообщения не доставляются:

Проверьте:
1. Канал в конфиге совпадает с каналом в MeshCore
2. Устройство подключено и работает:
meshcore-cli -s /dev/ttyUSB0 get_status
3. Reticulum запущен и работает

15.4. Ошибка: ModuleNotFoundError: No module named 'rns':

Решение:
pip3 install rns lxmf --upgrade

15.5. Ошибка: Permission denied при запуске:

Решение:
chmod +x RMesh_final.py start.sh run_bridge.sh start_terminal.sh

15.6. Ошибка: неправильная версия Python:

Решение:
# Установка Python 3.11
sudo apt install python3.11 python3.11-venv

# Использование правильной версии
python3.11 RMesh_final.py
